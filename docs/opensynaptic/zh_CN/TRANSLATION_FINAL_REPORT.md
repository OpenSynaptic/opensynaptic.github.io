## 💡 翻译方法论

### 分优先级策略

**第一优先级**（用户直接接触）
- 完整翻译所有 guides、plugins、features 文档
- 保留所有代码示例、命令和配置
- 精确翻译技术术语

**第二优先级**（架构参考）
- 完整翻译所有 architecture、api、core 文档
- 保留所有图表、表格和 API 签名
- 详细翻译技术细节

**第三优先级**（技术报告）
- 优先翻译发布说明（用户价值高）
- 选择性翻译关键技术报告
- 可选简化或保留内部文档

### 质量控制流程

1. **读取原文** → 完整获取内容
2. **分段翻译** → 保持结构和格式
3. **术语检查** → 使用建立的术语表
4. **格式验证** → 确保代码、表格、列表完整
5. **最终审查** → 中英对照无遗漏
